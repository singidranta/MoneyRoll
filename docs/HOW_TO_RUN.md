# 🚀 Как запустить MoneyRoll локально

> Пошаговая инструкция. Делай по пунктам.

---

## 1. Нужные программы

Открой **PowerShell** (или Terminal) и проверь, что установлено:

### Node.js 20+

Набери:
```
node --version
```

Если появилось `v20.x.x` или выше — всё ок. Если ошибка или старая версия — скачай с [nodejs.org](https://nodejs.org/) и установи обычным двойным кликом.

### pnpm (установщик пакетов)

Набери:
```
pnpm --version
```

Если ошибка «не найден» — установи:
```
npm install -g pnpm
```

---

## 2. Открой проект

В Cursor: **File → Open Folder** → выбери папку `MoneyRoll`.

Внизу экрана открой терминал: **View → Terminal** (или `Ctrl+~`).

---

## 3. Установи зависимости

В терминале, прямо в корне проекта:

```
pnpm install
```

Подожди 1-2 минуты, пока скачаются все библиотеки (Phaser, Express и т.д.).

В конце должно быть:
```
Done in XX.Xs
```
Без красных ошибок.

---

## 4. Запусти игру

```
pnpm dev
```

В терминале появится **два** процесса:

- `[SERVER]` — сервер на порту 3000 (`Server listening on http://localhost:3000`)
- `[CLIENT]` — игра на порту 5173 (`VITE vX.X.X ready in XXX ms`)

Браузер **сам не откроется**. Открой вручную:

```
http://localhost:5173
```

---

## 5. Что должно появиться

- Чёрный экран
- Зелёный квадратик в центре (это твой персонаж)
- Сетка-фон (мир)
- В консоли (F12 в браузере) → вкладка Console: `[MoneyRoll] WS connected`

Если всё так — **Этап 0 работает!** 🎉

---

## 6. Управление

| Клавиша | Действие |
|---|---|
| **WASD** | движение |
| **Стрелки ←↑↓→** | движение (альтернатива) |
| **F12 → Console** | открыть консоль браузера (для отладки) |
| **Ctrl+R** | перезагрузить страницу |

---

## 7. Если что-то не работает

### «Cannot find module ...» или красные ошибки в терминале
→ Не установлены зависимости. Выполни `pnpm install` ещё раз.

### Порт 3000 или 5173 занят
→ Закрой другое приложение, использующее этот порт. Или смени порт:
```
PORT=3001 pnpm dev:server
```

### Игра не обновляется после изменения кода
→ Должна обновляться автоматически. Если нет — **Ctrl+R** в браузере.

### Браузер показывает белый экран / ничего
→ Открой **F12 → Console** → посмотри красные ошибки → скинь мне.

### «ECONNREFUSED» в консоли
→ Сервер не запущен. Проверь, что в одном из терминалов видно `Server listening`.

### Другие ошибки
→ Открой **F12 → Console**, сделай скриншот красного текста — скинь мне, разберусь.

## 10. Режим редактора карты (`pnpm dev:editor`)

Отдельный режим для построения карты. Карта — **20×20 клеток**, тайл = **128×128 px**. Сохраняется сервером в `data/map.json`.

Запуск:
```
pnpm dev:editor
```

Открой **в браузере**:
```
http://localhost:5173/?mode=editor
```

> Режим (play или editor) определяется **URL-параметром `?mode=editor`**, а не env-переменной.
> Так работает одинаково в bash, PowerShell, cmd.exe и Git Bash. Если открыть просто `http://localhost:5173` — запустится play-режим.

### Управление

| Действие | Что произойдёт |
|---|---|
| **ЛКМ по клетке** | Ставит выбранный тайл или объект |
| **ПКМ** / режим «Ластик» | Удаляет объект из клетки; ластик также возвращает траву |
| **`R`** | Повернуть выбранный тайл/объект на 90° |
| **`WASD` / стрелки** | Двигать камеру по карте |
| **Space + ЛКМ** / средняя кнопка | Перетаскивать камеру |
| **`Ctrl+S`** | Принудительное сохранение (иначе — авто-сейв через 600мс) |

### Как поставить работу или недвижимость

1. Справа выбери **«Ставить объекты»**.
2. В выпадающем списке открой группу **«Работы»** и выбери курьера, лимонад или сортировку мусора; либо в группе **«Инвестиции»** выбери **«Точка покупки недвижимости»**.
3. Для недвижимости выбери ниже её тип: сарай, хрущёвка или пентхаус, затем кликни по клетке.
4. Нажми **Ctrl+S**. В play-режиме игрок подходит к цветной точке и нажимает **E**: работа выдаёт награду с кулдауном, недвижимость покупается и начинает приносить пассивный доход.

### Переключение режимов на лету

Из play → editor: добавь к URL `?mode=editor` (или `&mode=editor` если уже есть парам).
Из editor → play: убери `mode=editor` из URL.

### Где хранится карта

Сразу после сохранения — в `data/map.json`. Файл обновляется **на каждом изменении** (debounced 600мс) — можно редактировать даже без Ctrl+S. Если нужно сбросить карту — просто удали этот файл и перезапусти сервер.

> **Важно:** `pnpm dev` и `pnpm dev:editor` нельзя запускать одновременно — оба используют порты 5173 и 3000.

## 11. Игра по сети (Radmin VPN / LAN)

Игровой сервер слушает **все сетевые интерфейсы** (`0.0.0.0`). Vite тоже слушает все интерфейсы (`host: true`). Это нужно, чтобы друг зашёл по локальной сети или через Radmin VPN.

### При запуске сервера в консоли видишь:

```
[MoneyRoll][server] listening on http://0.0.0.0:3000
[MoneyRoll][server] SHARE URLS (give one to your friend):
  http://192.168.1.42:5173   (interface: Wi-Fi)        ← домашний Wi-Fi
  http://192.168.0.10:5173   (interface: Ethernet)     ← Ethernet
  http://26.12.34.56:5173    (interface: Radmin VPN)   ← Radmin
```

В HUD игры (правый верхний угол) — те же адреса списком. Дай другу **URL из списка**, который соответствует общей сети:

| Сценарий | Какой URL дать другу |
|---|---|
| Оба в одной Wi-Fi сети | URL интерфейса `Wi-Fi` |
| Через Radmin VPN | URL интерфейса `Radmin VPN` (виртуальный, обычно `26.x.x.x`) |
| Через Hamachi / подобное | Соответствующий виртуальный интерфейс |

### Если адресов нет вообще

- **Windows firewall**: разреши входящие на `node.exe` (порт 3000) и `vite` (порт 5173) — иначе блокирует.
- **Radmin VPN не активен**: запусти Radmin и подожди пока появится его интерфейс (виден в `ipconfig`).
- **Vite или сервер не перезапускал**: иногда после изменения конфигов нужно перезапустить — `Ctrl+C`, затем снова `pnpm dev`.

### Без Radmin: проверить между двумя ПК в одной сети

1. Хост: `pnpm dev` → смотри свой IP в консоли (`http://192.168.X.X:5173`).
2. Друг: открывает этот URL в браузере. Должен появиться второй зелёный квадратик.
3. Двигайся — двигаются оба, видят друг друга плавно (snapshot-интерполяция 20Hz).

## 12. Проверка сети из консоли (F12)

В игре открой **F12 → Console**, набери:
```
await fetch('/api/network').then(r => r.json())
```

Вернётся что-то вроде:
```json
{ "ips": [{"iface":"Wi-Fi","ip":"192.168.1.42"}, ...], "port": 5173 }
```
